Parking Slot Management System – 5 Week Timeline

Week 1 — Project Planning & Requirement Analysis

Goal: Understand the problem and prepare the project structure.
Tasks:
	Study the parking management problem and define system objectives.
	Identify key operations:
	Park vehicle
	Remove vehicle
	Search vehicle
	Display parking status
	Decide required data structures:
	Stack
	Heap (priority handling)
	Hash table (fast search)
	Design overall system workflow.
	Divide responsibilities among the five group members.
	Create the project repository.
	Prepare project documentation:
	Problem statement
	Project description
	Data structure justification

Output of Week 1
	Project plan
	System design
	Repository setup
	Documentation ready


Week 2 — Data Structure Design

Goal: Design and begin implementing the core data structures.
Tasks:
	Design the Stack structure
	Define attributes and operations.
	Design the Heap structure
	Define priority rules for vehicle handling.
	Design the Hash table structure
	Define hashing method for vehicle numbers.
	Implement the basic structure of each module.
	Write algorithms for:
	Insert operations
	Remove operations
	Search operations
	Perform small tests for each data structure.

Output of Week 2
	Working implementations of stack, heap, and hash structures.


Week 3 — Core Parking System Logic

Goal: Develop the main functionality of the parking system.
Tasks:
	Implement Park Vehicle operation
	Assign parking slots.
	Update system records.
	Implement Remove Vehicle operation
	Free parking slots.
	Update system records.
	Implement Search Vehicle operation
	Locate vehicles using hash table.
	Connect all data structures with the parking logic.
	Test parking and removal scenarios.

Output of Week 3
	Core system functionality working.


Week 4 — System Integration & Interface

Goal: Make the full system usable.
Tasks:
	Develop the user interaction system (menu-driven interface).
	Implement display functions:
	Display parking slots
	Display parked vehicles
	Display available slots
	Integrate all modules into a single system.
	Test the full workflow:
	Parking
	Removal
	Searching
	Displaying status

Output of Week 4
	Fully integrated system.


Week 5 — Testing, Debugging & Documentation

Goal: Finalize the project for submission.
Tasks:
	Perform complete system testing.
	Fix logical errors and bugs.
	Improve code readability and comments.
	Optimize variable naming and structure.
	Prepare final documentation:
	System explanation
	Data structures used
	Algorithm descriptions
	Verify GitHub commits and contributions.
	Prepare final submission files.

Output of Week 5
	Fully working project
	Clean and documented code
	Submission-ready system
